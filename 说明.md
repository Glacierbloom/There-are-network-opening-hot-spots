# 启动热点工程说明

## 项目简介

本项目是一个用于启动 Windows 网络共享热点的 C++ 控制台程序（仅当系统有可上网的网络连接时程序才会继续执行）。程序通过 WinRT API 获取当前网络连接配置，并尝试创建热点管理器，然后根据当前热点状态启动或提示用户热点已开启。

## 文件说明

- `启动热点.cpp`：项目主源文件，包含程序入口、WinRT 初始化、网络连接检测、热点管理以及控制台交互逻辑。
- `说明.md`：本说明文件。

## 编译方式

1. 使用 Visual Studio 或 `cl.exe` 编译。
2. 如果使用 `cl.exe`，可参考以下命令：

```bat
cl.exe /Zi /EHsc /std:c++17 启动热点.cpp /link windowsapp.lib ole32.lib oleaut32.lib
```

3. 生成的可执行文件通常位于当前目录或 `bin` 目录下。

## 运行说明

1. 运行前请确保系统支持 WinRT 网络热点相关 API。
2. 需要管理员权限或相应网络权限，具体取决于系统策略和热点管理器权限。
3. 程序启动后，会显示当前网络名称，并尝试启动热点。如果热点已经开启，则直接提示状态。
4. 程序结束前会等待最多 5 秒，或按任意键提前退出。

## 代码逻辑概述

- 设置控制台编码为 UTF-8，确保中文输出正常。
- 初始化 WinRT 运行环境为多线程公寓（MTA）。
- 获取当前互联网连接配置 `NetworkInformation::GetInternetConnectionProfile()`。
- 创建 `NetworkOperatorTetheringManager` 实例，并检查热点状态是否稳定。
- 如果热点未开启，则调用 `StartTetheringAsync().get()` 启动热点；如果已开启，则给出提示。
- 最终输出热点当前状态，并等待用户按键或超时退出。

## 注意事项

- 仅在支持 Windows 网络热点管理的设备上有效。
- 如果 `CreateFromConnectionProfile` 失败，程序会输出错误信息并退出。
- 如果热点状态长时间处于 `InTransition`，程序会在超时后停止操作。
